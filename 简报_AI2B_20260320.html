NVIDIA

NVIDIA Agent Toolkit 正式发布 —— OpenShell 安全运行时 + AI-Q 企业智能体蓝图

发布时间2026年3月16日（GTC 2026 主题演讲） 产品名称NVIDIA Agent Toolkit（含 OpenShell 运行时 + AI-Q 蓝图 + Nemotron 模型） 解决问题企业部署 AI Agent 时的安全管控、数据隐私与成本失控问题
NVIDIA 在 GTC 2026 上正式推出 Agent Toolkit，核心包含：① OpenShell ——开源 Agent 运行时沙箱，通过基于策略的护栏对"感知-推理-行动"全链路强制安全约束；② AI-Q 蓝图 ——基于 LangChain 构建的企业知识搜索 Agent，使用前沿模型编排 + Nemotron 执行，声称查询成本降低 50% 以上；③ 生态合作方已涵盖 Salesforce、Atlassian、ServiceNow、Siemens、IQVIA 等。已在 AWS / Azure / GCP / OCI 上线。
💡 对企业级 AI PC 的启示
  • OpenShell 将"Agent 沙箱"确立为行业基础设施标准，AI PC 端侧需同等级别的 Agent 运行时隔离能力，否则将在安全采购环节被排除
  • AI-Q 混合架构（大模型编排 + 小模型执行）降成本 50% 的数据，是 AI PC 本地推理分层调度的重要参考——可用本地小模型承担高频执行，云端大模型仅做规划
  • NVIDIA 通过 ISV 伙伴预装策略（Salesforce/Atlassian/ServiceNow），AI PC 的企业市场准入须同步考虑 Agent 框架与主流 SaaS 生态的深度绑定
📎 来源： prodSens.live（2026-03-19） · StockTitan / NVDA Press（2026-03-16）
动态 02 / 05
Manus

Manus 发布桌面端「我的电脑」—— 通用 AI Agent 从云端延伸至本地文件系统

发布时间2026年3月16日 产品名称Manus 桌面应用 · My Computer（我的电脑）功能 平台支持macOS、Windows 解决问题企业员工本地文件管理自动化、跨平台工作流打通（云 + 本地）、非技术用户使用门槛
Manus 通过桌面应用将 Cloud Agent 能力本地化：可直接在终端执行指令、读写本地文件、调用 Python / Node.js / Swift 等本地工具链；支持授权特定文件夹、每条终端指令需用户审批（可选"始终允许"）。典型场景包括批量整理发票、自动化代码构建、远程调度本地闲置 GPU 跑推理任务，以及云服务（Gmail / Slack）与本地数据的跨平台工作流。内部测试显示本地复杂任务完成效率提升显著（20 分钟完成原 3 小时任务）。
💡 对企业级 AI PC 的启示
  • "本地文件系统 + 工具链直接调用"正在成为 Agent 能力的标配要求，AI PC 须在硬件层面提供更低延迟的本地 Agent 执行环境（NPU 加速 + 沙箱隔离）
  • Manus 的"逐条审批"交互范式为企业级 AI PC 设计提供参考：企业场景需要可配置的权限粒度（只读/执行/始终允许），而非二元开关
  • 云+本地协同的混合 Agent 架构（云端规划 + 本地执行）印证了企业级 AI PC 的差异化方向：不是替代云，而是成为企业员工的本地执行节点
📎 来源： Manus 官方博客（2026-03-16） · OSChina（2026-03-17）

腾讯

腾讯电脑管家 18.0 发布「龙虾管家·AI 安全沙箱」—— 行业首个本地 AI Agent 一站式安全防护

发布时间2026年3月13日 产品名称腾讯电脑管家 18.0 · 龙虾管家·AI 安全沙箱 解决问题本地运行的 AI Agent（OpenClaw 等）引发的文件安全、支付欺诈、隐私泄露三大风险
腾讯电脑管家 18.0 版本正式推出行业首个"龙虾管家·AI 安全沙箱"，核心能力：① AI 安全沙箱隔离本地 Agent 执行环境，防止恶意操作扩散；② 首创可视化 AI 应用隐私监测，实时展示 AI 应用对剪贴板、摄像头等敏感权限的调用情况；③ 一键防护 OpenClaw 及其他本地 AI Agent 的异常支付行为。整体定位为"终端安全防护层"，嵌入企业员工电脑 Agent 全流程。
💡 对企业级 AI PC 的启示
  • 腾讯以"安全管家"切入本地 Agent 生态，预示着企业客户在采购 AI PC 时，"内置 Agent 安全防护"将成为刚性需求项，而非可选项
  • 可视化权限监控的交互设计（展示 AI 调用了哪些系统资源）是 AI PC 产品的差异化机会点：硬件级权限管控比软件层更具可信度
  • 腾讯龙虾生态快速扩张（QClaw + WorkBuddy + 龙虾管家三件套），AI PC 需警惕被平台封装为底层基础设施、失去用户感知
📎 来源： 腾讯云开发者社区（2026-03-18） · 新浪财经（2026-03-13）

360

360 发布「龙虾卫士」+ 安全龙虾 Box —— 周鸿祎称"以模治模"，10 分钟完成 Agent 安全部署

发布时间2026年3月14日（周鸿祎现场演示发布） 产品名称360龙虾卫士 + 360安全龙虾客户端 + 360安全龙虾Box（硬件） 解决问题OpenClaw 智能体部署门槛高（原需 6 小时配环境）、企业级 Agent 安全治理空白
360 集团在 3 月 14 日正式杀入"龙虾"赛道。360龙虾卫士采用"以模治模"设计理念，核心四层防护：① 虚拟化沙箱（WSL）隔离执行环境；② AI 安全引擎实时识别"技能投毒""提示词注入"等新型攻击；③ 最小权限原则限制 Agent 系统访问；④"人在回路"关键操作确认机制。已接入16 家国内主流大模型，支持 2.1 万+ 开源技能。360安全龙虾Box硬件终端将部署时间从 6 小时压缩至 10 分钟，软件基础功能免费，Token 算力按需付费（入门套餐 169 元）。周鸿祎金句："2026 年是龙虾之年，一人一龙虾、一企业多龙虾。"
💡 对企业级 AI PC 的启示
  • 360 将 Agent 部署简化为 10 分钟的硬件路径，直接对标 AI PC"开机即用"的核心价值主张，AI PC 须在零配置 Agent 部署上做到同等甚至更优的体验
  • "以模治模"的 AI 安全治理思路（用 AI 监控 AI）是企业级场景的前沿方向，AI PC 内置安全芯片若能支持此类推理，将构建独特的硬件级安全护城河
  • 169 元/Token 套餐的低门槛定价策略，提示 AI PC 2B 商业化需要提供"可核算、可预测"的成本模型，而非黑盒订阅
📎 来源： 澎湃新闻（2026-03-14） · 360龙虾卫士官网 · 新浪财经（2026-03-15）

Anthropic & Google

Claude Opus 4.6 上线（100 万 Token 上下文）+ Google ADK 重大更新，企业 Agent 框架竞赛白热化

发布时间2026年3月（本期覆盖窗口内） AnthropicClaude Opus 4.6：100 万 Token 上下文窗口，Agent 编码与长周期任务大幅提升，含"快速模式" GoogleAgent Development Kit（ADK）重大更新：新增 TypeScript SDK + AgentTeam API（多 Agent 协调）+ Gemini 2.5 Pro 深度集成 解决问题企业级 Agent 在长文档处理、跨团队协作、多模态实时交互方面的能力瓶颈
Anthropic 发布 Claude Opus 4.6，百万 Token 上下文可处理整个代码库或多日对话历史，Agent 编码场景表现显著改进；Thomson Reuters、eSentire、Doctolib、L'Oréal 等企业已披露具体 ROI 数据（如 eSentire 威胁分析从 5 小时压缩至 7 分钟，AI 与专家一致性达 95%）。Google ADK 同步更新 TypeScript SDK、推出 AgentTeam API 支持多 Agent 协调，与 Vertex AI + Gemini 2.5 Pro 深度整合。两家均在推动 Agent 框架标准化——Google A2A 协议与 Anthropic MCP 正在形成互联互通标准的核心竞争格局。
💡 对企业级 AI PC 的启示
  • 百万 Token 上下文意味着本地 Agent 将需处理越来越大的工作记忆，AI PC 的内存配置策略（LPDDR + 统一内存架构）需与模型上下文需求趋势对齐
  • 多 Agent 协调（AgentTeam API）正成为企业级标配，AI PC 需考虑作为团队中某个专属执行 Agent 的节点定位，而非单机孤岛
  • A2A 与 MCP 标准化进程直接影响 AI PC 的生态接入选择，需尽早明确支持哪套协议，避免被生态边缘化
